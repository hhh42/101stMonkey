Dec 11, 2014 · 2 minute read

itSimple

itSimple is a very ambitious project to take the world of PDDL editors from basically simple text editing with nearly no error output other than failure or full solution.
Having fought writing PDDL over the years with no tools it was a welcome find. Though skeptical of it’s ability to satisfy the claims it made I’m beginning to be a believer.
It DOES feel like the tool limits you with the expressiveness of PDDL, I’ve come to embrace the patterns it forces you to do. It’s certainly kept me out of trouble that I have historically gotten into.

Multiple approached

itSimple allows you to express the same thing in many way that end up with effectively (or exactly) the same PDDL output. The Finite State Machine metaphor the force you to work within is a good visual mapping to the model your working with. The tool itself is rather buggy but I’ve learned to work around most of them.

Don’t edit the XML file

Tempting as it may sound. Editing the input file has yielded me some more capabilities but very quickly you break parts of the app that you’ll miss.

OCML sucks

Even when you manage to find the pdf file on the site and decode the inscrutable doc… and work around the differences in the doc and the tool itself… You end up wishing you could just write the PDDL that it generates and call it a day. My approach has been one of a minimalist, use the few patterns I know that work and pretend that most don’t exists. I’m reserved myself to being “stuck” in PDDL 2.1 but I can’t seem to find a stable enough planner for any of the structures beyond that so it’s I’m stuck for a few reasons. Frankly I don’t feel that it’s a limitation.

Static analysis beats soft constraints?

It seems I won’t be getting soft contraints so I’m going to play games with various cost functions and maybe even statically evaluate the plan after the fact if I need too. It sure seems like that I’ll have to do that for that contraint satisfaction system anyway.. Since I’m computing multiple solutions anyway for finding the unique solutions so it’s not much of a leap to evaluate it’s cost at that point and see if that can be used to guide it… or allow the user to select preferences that manifest as soft contraints… Anyway.. should probably move that to a CSP entry
