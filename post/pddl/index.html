The attraction to a true declarative language

I was shocked how quickly one could start using PDDL. Since there are barely any keywords to know in the langauge the result is you can code as soon as you get the mechanism.

Finite State Machines

Using Finiste State Machines to model the behavior of the objects in the world and then using itSimple to generate PDDL via its internal XML representation allows you to not write the actions explicitly. The practicle result is you have to be extremely aware of guard clauses and the like in the FSM and as part of debugging your constantly checking the generated conditions in the info box within the itSimple interface. Also I seem to be checking the PDDL generated output tabs in itSimple as well when you have malformed precondtion or the like. I’ve been stumped by many a (and ()) precondition or effect which doesn’t fly in most planners.
